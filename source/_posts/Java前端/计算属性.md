计算属性是基于它们的依赖进行缓存的。计算属性只有在它的相关依赖发生改变时才会重新求值。这就意味着只要 message 还没有发生改变，多次访问 reversedMessage 计算属性会立即返回之前的计算结果，而不必再次执行函数。

这也同样意味着下面的计算属性将不再更新，因为 Date.now() 不是响应式依赖：
```html
computed: {
  now: function () {
    return Date.now()
  }
}
```
```javascript
computed: {
  now: function () {
    return Date.now()
  }
}
```
